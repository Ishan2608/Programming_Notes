Installation & Project Structure

FastAPI is a modern, high-performance Python web framework built on top of Starlette (ASGI) and Pydantic (data validation). It generates OpenAPI docs automatically and is one of the fastest Python frameworks available — on par with Node.js and Go in benchmarks.

Key selling points vs Flask/Django

  • Automatic docs — Swagger UI at /docs, ReDoc at /redoc, generated from type hints with zero config
  • Native async — Built on ASGI; async def route handlers work out of the box
  • Pydantic validation — Request/response data is validated and serialised automatically
  • Type-hint driven — Parameters, bodies, query strings all inferred from Python type hints

Install

uvicorn is the ASGI server used to run FastAPI applications. python-multipart is required for form and file upload support.

Recommended project layout

app/main.pyApp instance, startup/shutdown events
app/routers/One file per domain (users.py, items.py …)
app/models/SQLAlchemy ORM models
app/schemas/Pydantic request/response schemas
app/dependencies.pyShared dependency functions (DB session, auth …)
app/crud.pyDatabase query functions
⚡ vs Flask
FastAPI is ASGI, Flask is WSGI. You can use sync def routes, but async unlocks real concurrency. Use uvicorn (or gunicorn + uvicorn workers) — never Flask's dev server.
code
# Install FastAPI and the Uvicorn ASGI server
pip install fastapi uvicorn[standard]

# Additional common packages
pip install python-multipart   # form data / file uploads
pip install pydantic[email]    # email validation support
pip install sqlalchemy         # database ORM
pip install python-jose[cryptography]  # JWT tokens
pip install passlib[bcrypt]    # password hashing
pip install httpx              # async HTTP client (also used by TestClient)

App Instance & Running

The FastAPI() class is your application. It extends Starlette, so any Starlette feature works here too. Configuration passed to the constructor is reflected directly in the auto-generated OpenAPI schema.

FastAPI() constructor parameters

ParameterTypeDescription
titlestrAPI title shown in Swagger UI. Default: "FastAPI"
descriptionstrMarkdown-supported description in the docs.
versionstrAPI version string. Default: "0.1.0"
docs_urlstr | NonePath to Swagger UI. Set to None to disable. Default: "/docs"
redoc_urlstr | NonePath to ReDoc UI. Default: "/redoc"
openapi_urlstr | NonePath to raw OpenAPI JSON. Default: "/openapi.json"
lifespancallableAsync context manager for startup/shutdown events (replaces deprecated on_event).

Running the server

uvicorn app.main:app --reloadDev mode with auto-reload
uvicorn app.main:app --workers 4Production multi-worker
--host 0.0.0.0 --port 8080Bind address / port
--log-level warningReduce log noise in prod

# app/main.py — minimal but production-conscious boilerplate
from contextlib import asynccontextmanager
from fastapi import FastAPI


@asynccontextmanager
async def lifespan(app: FastAPI):
    # ── Startup: runs before the server starts accepting requests ──
    print("Starting up: connect DB, warm caches, load models …")
    yield
    # ── Shutdown: runs after the last request completes ───────────
    print("Shutting down: close DB connections, flush queues …")


app = FastAPI(
    title="My API",
    description="A well-structured FastAPI service",
    version="1.0.0",
    lifespan=lifespan,
)


@app.get("/")
async def root():
    return {"message": "Hello, World"}

Path Operations & HTTP Methods

A path operation is a function decorated with an HTTP method decorator. FastAPI maps decorators to HTTP methods and registers routes automatically. The function return value is serialised to JSON.

Available decorators

GET@app.get(path) — read data
POST@app.post(path) — create
PUT@app.put(path) — full replace
PATCH@app.patch(path) — partial update
DELETE@app.delete(path) — remove
OPTIONS / HEAD@app.options / @app.head

Decorator parameters

ParameterTypeDescription
pathstrURL path string. Use {param} for path parameters. Required.
response_modelPydantic modelSchema used to filter and validate the response. Strips extra fields automatically.
status_codeintDefault HTTP status code for success responses. Default: 200
tagslist[str]Groups routes in Swagger UI. e.g. ["users"]
summarystrShort description shown in Swagger UI.
descriptionstrMarkdown-supported longer description.
deprecatedboolMarks the route as deprecated in the docs.
include_in_schemaboolSet False to hide from OpenAPI docs (e.g. internal health endpoints).
💡 Tip
FastAPI executes async def routes in the event loop and def routes in a separate thread pool — so sync routes don't block the loop. Prefer async def for I/O-bound work (DB, HTTP calls) and def is fine for CPU-bound or trivial routes.
code
from fastapi import FastAPI

app = FastAPI()

# ── GET ───────────────────────────────────────────────────────────────────────
@app.get("/items", tags=["items"], summary="List all items")
async def list_items():
    return [{"id": 1, "name": "Widget"}]

# ── POST ──────────────────────────────────────────────────────────────────────
@app.post("/items", status_code=201, tags=["items"])
async def create_item():
    return {"id": 2, "name": "New item"}

# ── PUT ───────────────────────────────────────────────────────────────────────
@app.put("/items/{item_id}", tags=["items"])
async def replace_item(item_id: int):
    return {"id": item_id, "updated": True}

# ── DELETE ────────────────────────────────────────────────────────────────────
@app.delete("/items/{item_id}", status_code=204, tags=["items"])
async def delete_item(item_id: int):
    return   # 204 No Content — return nothing

# ── Sync route (runs in thread pool, doesn't block event loop) ────────────────
@app.get("/sync-ok", include_in_schema=False)
def sync_route():
    return {"sync": True}

Path & Query Parameters

FastAPI infers parameter source from where it appears: a name matching a {placeholder} in the path is a path parameter; any other function argument is a query parameter. No decorators or extra config needed.

Path Parameters

Declare in the path string with {name} and match with the function argument. Use Path() to add validation and docs metadata.

Query Parameters

Any argument not in the path is automatically a query parameter. Optional ones use = None or Optional[T]. Use Query() for constraints.

Path() and Query() constraints

ArgumentApplies toDescription
defaultQueryDefault value if not supplied. Use ... (Ellipsis) to make it required.
ge / lePath, QueryGreater-than-or-equal / less-than-or-equal (numeric).
gt / ltPath, QueryStrictly greater / less than.
min_length / max_lengthQueryString length constraints.
patternQueryRegex pattern the string must match.
aliasQueryDifferent name in the URL (e.g. alias="item-id" for item_id).
title / descriptionBothShown in Swagger UI docs.
deprecatedBothMarks parameter as deprecated in docs.
code
from typing import Annotated
from fastapi import FastAPI, Path, Query

app = FastAPI()

# ── Path parameter — type-annotated, auto-validated ──────────────────────────
@app.get("/users/{user_id}")
async def get_user(
    user_id: Annotated[int, Path(ge=1, title="User ID", description="Must be a positive integer")]
):
    return {"user_id": user_id}

# ── Query parameters — optional, with defaults ────────────────────────────────
@app.get("/items")
async def list_items(
    skip:  int        = Query(0,   ge=0),
    limit: int        = Query(20,  ge=1, le=100),
    q:     str | None = Query(None, min_length=2, max_length=50),
):
    # GET /items?skip=0&limit=20&q=widget
    return {"skip": skip, "limit": limit, "q": q}

# ── Multiple path params + query params together ──────────────────────────────
@app.get("/users/{user_id}/orders/{order_id}")
async def get_order(
    user_id:  int,
    order_id: int,
    include_items: bool = False,   # query param with default
):
    return {"user_id": user_id, "order_id": order_id, "items": include_items}

# ── List query param — ?tag=a&tag=b&tag=c ─────────────────────────────────────
@app.get("/search")
async def search(tags: list[str] = Query(default=[])):
    return {"tags": tags}

Request Body & Pydantic Models

Pydantic v2

Any parameter typed as a Pydantic BaseModel subclass is automatically parsed from the JSON request body. Pydantic v2 (used by FastAPI ≥ 0.100) is significantly faster than v1 and uses model_config instead of inner Config classes.

BaseModel essentials

  • Fields without defaults are required. Fields with defaults are optional.
  • Use Field() to add validation, docs metadata, and examples.
  • Models can be nested — declare a field with another model type.
  • model_config = ConfigDict(from_attributes=True) enables ORM mode (reading from SQLAlchemy objects).

Body() for extra control

Use Body() to embed multiple body parameters, mark one as the sole body, or add metadata. For simple cases, just type-hint with the Pydantic model.

💡 Schema separation pattern
A common pattern: ItemCreate (input, no id), ItemUpdate (all optional), ItemResponse (output with id). Never reuse one model for both input and output — they have different field requirements.
code
from pydantic import BaseModel, Field, EmailStr, ConfigDict
from typing import Annotated
from fastapi import FastAPI, Body

app = FastAPI()

# ── Input schema (no id — DB assigns it) ─────────────────────────────────────
class ItemCreate(BaseModel):
    name:        str            = Field(..., min_length=1, max_length=100, examples=["Laptop"])
    description: str | None    = Field(None, max_length=500)
    price:       float          = Field(..., gt=0, description="Must be greater than 0")
    tax:         float          = Field(0.0, ge=0)
    tags:        list[str]      = []

# ── Partial update schema — all fields optional ───────────────────────────────
class ItemUpdate(BaseModel):
    name:        str | None    = None
    description: str | None    = None
    price:       float | None  = None

# ── Response schema — includes DB-generated id ────────────────────────────────
class ItemResponse(BaseModel):
    model_config = ConfigDict(from_attributes=True)   # ORM mode
    id:          int
    name:        str
    price:       float
    price_with_tax: float | None = None

# ── Nested model ──────────────────────────────────────────────────────────────
class Address(BaseModel):
    street: str
    city:   str
    zip:    str

class UserCreate(BaseModel):
    email:   EmailStr
    address: Address    # nested — FastAPI validates recursively

# ── Routes ────────────────────────────────────────────────────────────────────
@app.post("/items", response_model=ItemResponse, status_code=201)
async def create_item(item: ItemCreate):
    price_with_tax = item.price + item.price * item.tax
    return {"id": 1, "name": item.name, "price": item.price, "price_with_tax": price_with_tax}

@app.patch("/items/{item_id}", response_model=ItemResponse)
async def update_item(item_id: int, item: ItemUpdate):
    # item.model_dump(exclude_unset=True) — only fields the client sent
    update_data = item.model_dump(exclude_unset=True)
    return {"id": item_id, "name": "Updated", "price": 9.99, **update_data}

Response Models & Status Codes

response_model in the decorator tells FastAPI what shape to return. Even if your function returns a dict with extra keys, the response model filters them out. This prevents accidentally leaking internal fields (e.g. password hashes).

Controlling serialisation

  • response_model_exclude_unset=True — omit fields not explicitly set
  • response_model_exclude_none=True — omit None fields from response
  • response_model_include={"id", "name"} — whitelist fields
  • response_model_exclude={"internal_flag"} — blacklist fields

JSONResponse and custom responses

Return a JSONResponse directly when you need full control over headers, cookies, or status code at runtime. Use Response subclasses for non-JSON formats.

Common status codes — use status module, not raw ints

200HTTP_200_OK — default GET/PUT/PATCH
201HTTP_201_CREATED — POST creates resource
204HTTP_204_NO_CONTENT — DELETE, no body
400HTTP_400_BAD_REQUEST — malformed input
401HTTP_401_UNAUTHORIZED — not authenticated
403HTTP_403_FORBIDDEN — authenticated, no permission
404HTTP_404_NOT_FOUND — resource missing
422HTTP_422_UNPROCESSABLE_ENTITY — validation error (auto)
code
from fastapi import FastAPI
from fastapi.responses import JSONResponse, PlainTextResponse, RedirectResponse
from fastapi import status
from pydantic import BaseModel

app = FastAPI()

class UserPublic(BaseModel):
    id:    int
    email: str

class UserInternal(UserPublic):
    hashed_password: str   # filtered out by response_model

# ── response_model filters out hashed_password automatically ──────────────────
@app.get("/users/{id}", response_model=UserPublic)
async def get_user(id: int):
    return UserInternal(id=id, email="user@example.com", hashed_password="$2b$...")

# ── Runtime status code — determined after the function runs ──────────────────
@app.get("/items/{id}")
async def get_item(id: int):
    item = None  # simulate DB miss
    if not item:
        return JSONResponse(status_code=status.HTTP_404_NOT_FOUND,
                             content={"detail": "Item not found"})
    return item

# ── Exclude None fields from response ─────────────────────────────────────────
class Item(BaseModel):
    id:          int
    name:        str
    description: str | None = None

@app.get("/items-clean/{id}", response_model=Item, response_model_exclude_none=True)
async def get_item_clean(id: int):
    return Item(id=id, name="Widget")  # description omitted from JSON response

# ── Non-JSON responses ────────────────────────────────────────────────────────
@app.get("/health", response_class=PlainTextResponse, include_in_schema=False)
async def health():
    return "OK"

@app.get("/old-url")
async def redirect():
    return RedirectResponse(url="/new-url", status_code=301)

Data Validation & Field Types

Pydantic v2 validates data at parse time and raises structured errors. FastAPI catches these and returns a 422 Unprocessable Entity with full details — which field failed, what value was received, what was expected.

Common Pydantic field types

TypeImportValidates
EmailStrpydanticValid email format
HttpUrlpydanticValid HTTP/HTTPS URL
AnyUrlpydanticAny valid URL
UUIDuuidUUID string or object
datetimedatetimeISO 8601 datetime string
EnumenumValue must be a valid enum member
Literal["a","b"]typingExact string values only
constr(pattern=...)pydanticString matching regex
confloat(ge=0, le=1)pydanticFloat in range [0,1]

Custom validators (Pydantic v2)

Use @field_validator for per-field logic and @model_validator for cross-field validation. Replace Pydantic v1's @validator.

⚠ Pydantic v1 → v2 migration
If migrating from FastAPI < 0.100: @validator → @field_validator, class Config → model_config = ConfigDict(), .dict() → .model_dump(), .json() → .model_dump_json().
code
from pydantic import BaseModel, Field, EmailStr, HttpUrl, field_validator, model_validator
from enum import Enum
from datetime import datetime
from uuid import UUID
from typing import Literal

# ── Enum field ─────────────────────────────────────────────────────────────────
class Status(str, Enum):
    active   = "active"
    inactive = "inactive"
    pending  = "pending"

# ── Rich validation model ──────────────────────────────────────────────────────
class UserCreate(BaseModel):
    id:          UUID                          # auto-parsed from string
    email:       EmailStr
    website:     HttpUrl | None   = None
    status:      Status            = Status.active
    role:        Literal["admin", "user", "guest"] = "user"
    created_at:  datetime           = Field(default_factory=datetime.utcnow)
    age:         int               = Field(..., ge=0, le=150)
    username:    str               = Field(..., pattern=r"^[a-zA-Z0-9_]{3,30}$")
    password:    str               = Field(..., min_length=8)
    confirm_pw:  str

    # ── Field validator — runs on a single field ──────────────────────────────
    @field_validator("username")
    @classmethod
    def username_lowercase(cls, v: str) -> str:
        return v.lower()

    # ── Model validator — cross-field checks ──────────────────────────────────
    @model_validator(mode="after")
    def passwords_match(self) -> "UserCreate":
        if self.password != self.confirm_pw:
            raise ValueError("Passwords do not match")
        return self

Dependency Injection

FastAPI's DI system is one of its most powerful features. Dependencies are callables (functions or classes) declared with Depends(). They can have their own dependencies, forming a tree. FastAPI resolves and caches them per-request automatically.

Why use dependencies?

  • Share logic: authentication checks, DB sessions, pagination params
  • Testable: swap real deps with mocks in tests via app.dependency_overrides
  • Clean routes: route functions stay thin, logic lives in reusable deps

Dependency types

TypeWhen to use
Regular functionStateless logic — parse pagination, common headers
Class with __init__Configurable deps — parameterisable permission checker
async generator (yield)Resource management — DB session open/close, lock acquire/release
Route-level dependencies=[]Run deps for side-effects only (auth, rate limiting) without injecting return value
code
from fastapi import FastAPI, Depends, HTTPException, status
from typing import Annotated

app = FastAPI()

# ── Simple function dependency ─────────────────────────────────────────────────
def pagination(skip: int = 0, limit: int = 20) -> dict:
    return {"skip": skip, "limit": limit}

@app.get("/items")
async def list_items(page: Annotated[dict, Depends(pagination)]):
    return page

# ── Class-based dep — configurable ────────────────────────────────────────────
class RoleChecker:
    def __init__(self, allowed_roles: list[str]):
        self.allowed_roles = allowed_roles

    def __call__(self, current_user: dict = {"role": "user"}):
        if current_user["role"] not in self.allowed_roles:
            raise HTTPException(status_code=403, detail="Forbidden")

allow_admin = RoleChecker(["admin"])

@app.delete("/users/{id}", dependencies=[Depends(allow_admin)])
async def delete_user(id: int):
    return {"deleted": id}

# ── Generator dependency — DB session (yield pattern) ─────────────────────────
from sqlalchemy.orm import Session

def get_db() -> Session:
    db = SessionLocal()  # create session (see Section 12)
    try:
        yield db          # inject into route
    finally:
        db.close()       # always close, even on exception

DB = Annotated[Session, Depends(get_db)]  # reusable type alias

@app.get("/users")
async def get_users(db: DB):
    return db.query(User).all()

# ── Override in tests ─────────────────────────────────────────────────────────
def override_get_db():
    yield test_db_session

app.dependency_overrides[get_db] = override_get_db  # swap for tests

Routers & App Structure

APIRouter is FastAPI's equivalent of Flask Blueprints or Express Routers. Each domain (users, items, orders) gets its own router file. The main app includes them with a prefix and shared metadata.

APIRouter() parameters

ParameterDescription
prefixURL prefix added to all routes in this router. e.g. "/users"
tagsSwagger UI group label for all routes. e.g. ["users"]
dependenciesDependencies applied to every route in this router (e.g. auth check).
responsesCommon response schemas (e.g. {404: {"description": "Not found"}}).
💡 Pattern
Declare your router in each domain file. Import and include all routers in main.py. Keep main.py thin — only app creation, middleware, and includes.
code
# ── app/routers/users.py ──────────────────────────────────────────────────────
from fastapi import APIRouter, Depends, HTTPException

router = APIRouter(
    prefix="/users",
    tags=["users"],
    responses={404: {"description": "User not found"}},
)

@router.get("/")                          # resolves to GET /users/
async def list_users(): ...

@router.get("/{user_id}")               # resolves to GET /users/{user_id}
async def get_user(user_id: int): ...

@router.post("/", status_code=201)      # resolves to POST /users/
async def create_user(): ...


# ── app/routers/items.py ──────────────────────────────────────────────────────
items_router = APIRouter(prefix="/items", tags=["items"])

@items_router.get("/")
async def list_items(): ...


# ── app/main.py — include all routers ─────────────────────────────────────────
from fastapi import FastAPI
from app.routers import users, items
from app.dependencies import verify_api_key   # shared auth dep

app = FastAPI()

# Apply auth to all /admin routes; open access for /users
app.include_router(users.router)
app.include_router(items.items_router)
app.include_router(
    admin_router,
    prefix="/admin",
    dependencies=[Depends(verify_api_key)],
)

Middleware

Middleware intercepts every request before it reaches a route and every response before it leaves. Use it for cross-cutting concerns: CORS, request logging, timing, compression, rate limiting.

Built-in middleware

  • CORSMiddleware — Cross-Origin Resource Sharing headers
  • GZipMiddleware — Compress responses above a size threshold
  • HTTPSRedirectMiddleware — Force HTTPS
  • TrustedHostMiddleware — Reject requests with wrong Host header

CORS configuration

ParameterDescription
allow_originsList of allowed origins. Use ["*"] for public APIs only.
allow_credentialsSet True to allow cookies/auth headers. Cannot combine with allow_origins=["*"].
allow_methodsHTTP methods allowed. Default: ["GET"]
allow_headersHeaders allowed in preflight. ["*"] to allow all.
max_ageSeconds to cache preflight response. Default: 600
code
import time
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware
from fastapi.middleware.gzip import GZipMiddleware
from starlette.middleware.base import BaseHTTPMiddleware

app = FastAPI()

# ── CORS — configure before other middleware ───────────────────────────────────
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.com", "http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# ── GZip compression for large responses ──────────────────────────────────────
app.add_middleware(GZipMiddleware, minimum_size=1000)

# ── Custom middleware — request timing + correlation ID ───────────────────────
class TimingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        start = time.perf_counter()
        response = await call_next(request)
        duration = time.perf_counter() - start
        response.headers["X-Process-Time"] = f"{duration:.4f}s"
        return response

app.add_middleware(TimingMiddleware)

# ── Middleware via decorator (Starlette style) ────────────────────────────────
@app.middleware("http")
async def log_requests(request: Request, call_next):
    print(f"→ {request.method} {request.url}")
    response = await call_next(request)
    print(f"← {response.status_code}")
    return response

Authentication & Security

FastAPI provides security scheme classes in fastapi.security that integrate with the OpenAPI docs — the Swagger UI shows an Authorize button automatically. The most common pattern is JWT Bearer tokens via OAuth2PasswordBearer.

Security schemes

OAuth2PasswordBearerBearer JWT token in Authorization header
OAuth2PasswordRequestFormusername+password form body (login endpoint)
HTTPBasicBasic auth (username:password base64)
APIKeyHeaderAPI key in a request header
APIKeyQueryAPI key as query parameter
APIKeyCookieAPI key in cookie

JWT token flow

  • Client sends POST /token with credentials → server returns access_token
  • Client sends Authorization: Bearer <token> on subsequent requests
  • Dependency decodes token, fetches user, raises 401 if invalid/expired
⚠ Never store passwords in plaintext
Use passlib with bcrypt. Never use MD5 or SHA1 for passwords. Always use secrets.compare_digest() for constant-time string comparison to prevent timing attacks.
code
from datetime import datetime, timedelta
from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from pydantic import BaseModel

# ── Config ────────────────────────────────────────────────────────────────────
SECRET_KEY = "change-this-to-a-long-random-secret-in-production"
ALGORITHM  = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

pwd_ctx = CryptContext(schemes=["bcrypt"], deprecated="auto")
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

app = FastAPI()

# ── Token schema ──────────────────────────────────────────────────────────────
class Token(BaseModel):
    access_token: str
    token_type:   str

# ── Helpers ───────────────────────────────────────────────────────────────────
def verify_password(plain: str, hashed: str) -> bool:
    return pwd_ctx.verify(plain, hashed)

def create_access_token(data: dict, expires_delta: timedelta | None = None) -> str:
    payload = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(minutes=15))
    payload.update({"exp": expire})
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

# ── Login endpoint — returns JWT ──────────────────────────────────────────────
@app.post("/token", response_model=Token)
async def login(form: OAuth2PasswordRequestForm = Depends()):
    user = fake_authenticate(form.username, form.password)  # your DB lookup
    if not user:
        raise HTTPException(status_code=401, detail="Incorrect credentials",
                            headers={"WWW-Authenticate": "Bearer"})
    token = create_access_token({"sub": user.email},
                                 timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES))
    return {"access_token": token, "token_type": "bearer"}

# ── Reusable auth dependency ───────────────────────────────────────────────────
async def get_current_user(token: str = Depends(oauth2_scheme)):
    credentials_exc = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        email: str = payload.get("sub")
        if email is None: raise credentials_exc
    except JWTError:
        raise credentials_exc
    return get_user_from_db(email)   # your DB lookup

@app.get("/me")
async def read_me(current_user = Depends(get_current_user)):
    return current_user

Database Integration (SQLAlchemy)

FastAPI has no built-in ORM — you choose. SQLAlchemy 2.0+ with its new-style mapped_column API is the most common choice. Each request gets its own DB session, managed by a generator dependency (see Section 8).

Setup checklist

  • Engine — single instance, created once at startup
  • SessionLocal — factory that creates per-request sessions
  • Base — all ORM models extend this
  • get_db() — yield dependency that opens/closes sessions

Database URL formats

PostgreSQLpostgresql+psycopg2://user:pw@host/db
MySQLmysql+pymysql://user:pw@host/db
Async Postgrespostgresql+asyncpg://user:pw@host/db
⚠ SQLite + async
SQLite's default driver is synchronous. For async FastAPI with SQLite in dev, use aiosqlite with create_async_engine. For production, use PostgreSQL with asyncpg.

Async & Background Tasks

async

BackgroundTasks runs a function after the HTTP response is sent — without blocking the client. Use it for fire-and-forget work: sending emails, writing audit logs, triggering webhooks. For heavy workloads, use Celery or ARQ instead.

Async patterns

  • Use asyncio.gather() to run multiple coroutines concurrently
  • Use httpx.AsyncClient for async outbound HTTP calls — never requests in async routes
  • Use asyncio.create_task() for truly detached fire-and-forget
💡 When to use each
BackgroundTasks — simple post-response work, no retry logic needed. Celery / ARQ — retries, scheduling, distributed workers, long-running jobs.
code
import asyncio
import httpx
from fastapi import FastAPI, BackgroundTasks

app = FastAPI()

# ── Background task function (can be sync or async) ───────────────────────────
def send_welcome_email(email: str, username: str):
    # runs after response is sent — blocking here doesn't affect the client
    print(f"Sending welcome email to {email}…")

@app.post("/users", status_code=201)
async def create_user(background_tasks: BackgroundTasks):
    user = {"id": 1, "email": "user@example.com", "username": "alice"}
    background_tasks.add_task(send_welcome_email, user["email"], user["username"])
    return user   # response sent immediately; email fires after

# ── Concurrent async calls — fetch multiple APIs at once ──────────────────────
@app.get("/aggregate")
async def aggregate():
    async with httpx.AsyncClient() as client:
        users_task  = client.get("https://api.example.com/users")
        orders_task = client.get("https://api.example.com/orders")
        users_r, orders_r = await asyncio.gather(users_task, orders_task)
    return {"users": users_r.json(), "orders": orders_r.json()}

# ── Run CPU-heavy code in thread pool (don't block the event loop) ────────────
import hashlib
from fastapi.concurrency import run_in_threadpool

@app.post("/hash")
async def hash_data(data: str):
    result = await run_in_threadpool(hashlib.sha256, data.encode())
    return {"hash": result.hexdigest()}

File Uploads & Static Files

Use UploadFile for file uploads — it wraps the file with metadata and exposes an async read interface. File() is the equivalent of Form() for file fields. Requires python-multipart installed.

UploadFile attributes

Attribute / MethodDescription
filenameOriginal filename from the client (str)
content_typeMIME type e.g. "image/jpeg"

await seek(offset)Move file pointer

code
from fastapi import FastAPI, File, UploadFile, Form, HTTPException
from fastapi.staticfiles import StaticFiles
import shutil, pathlib

app = FastAPI()

# ── Serve static files (images, CSS, JS) ──────────────────────────────────────
app.mount("/static", StaticFiles(directory="static"), name="static")

UPLOAD_DIR = pathlib.Path("uploads")
UPLOAD_DIR.mkdir(exist_ok=True)
ALLOWED_TYPES = {"image/jpeg", "image/png", "application/pdf"}
MAX_SIZE = 5 * 1024 * 1024  # 5 MB

# ── Single file upload ────────────────────────────────────────────────────────
@app.post("/upload")
async def upload_file(file: UploadFile = File(...)):
    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(400, detail=f"File type not allowed: {file.content_type}")

    contents = await file.read()
    if len(contents) > MAX_SIZE:
        raise HTTPException(413, detail="File too large (max 5 MB)")

    dest = UPLOAD_DIR / file.filename
    with open(dest, "wb") as f:
        f.write(contents)

    await file.close()
    return {"filename": file.filename, "size": len(contents)}

# ── File + form fields together ───────────────────────────────────────────────
@app.post("/upload-with-meta")
async def upload_with_meta(
    file:        UploadFile = File(...),
    description: str        = Form(...),
    is_public:   bool       = Form(False),
):
    return {"filename": file.filename, "description": description, "public": is_public}

# ── Multiple files ────────────────────────────────────────────────────────────
@app.post("/upload-many")
async def upload_many(files: list[UploadFile] = File(...)):
    return [{"filename": f.filename, "type": f.content_type} for f in files]

WebSockets

FastAPI supports WebSockets natively via Starlette. Use @app.websocket(path) and accept a WebSocket parameter. Dependencies work with WebSocket routes too.

WebSocket methods

MethodDescription
await ws.accept()Complete the WebSocket handshake. Must call before sending/receiving.
await ws.receive_text()Receive next text message (blocks until message arrives)
await ws.receive_bytes()Receive next binary message
await ws.receive_json()Receive and parse JSON message
await ws.send_text(data)Send a text message
await ws.send_bytes(data)Send binary data
await ws.send_json(data)Serialise dict and send as JSON
await ws.close(code)Close the connection. Default code: 1000 (normal closure)
code
from fastapi import FastAPI, WebSocket, WebSocketDisconnect

app = FastAPI()

# ── Simple echo WebSocket ─────────────────────────────────────────────────────
@app.websocket("/ws")
async def websocket_endpoint(ws: WebSocket):
    await ws.accept()
    try:
        while True:
            data = await ws.receive_text()
            await ws.send_text(f"Echo: {data}")
    except WebSocketDisconnect:
        print("Client disconnected")

# ── Connection manager — broadcast to multiple clients ────────────────────────
class ConnectionManager:
    def __init__(self):
        self.active: list[WebSocket] = []

    async def connect(self, ws: WebSocket):
        await ws.accept()
        self.active.append(ws)

    def disconnect(self, ws: WebSocket):
        self.active.remove(ws)

    async def broadcast(self, message: str):
        for connection in self.active:
            await connection.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws/chat/{room_id}")
async def chat(ws: WebSocket, room_id: int):
    await manager.connect(ws)
    try:
        while True:
            data = await ws.receive_text()
            await manager.broadcast(f"Room {room_id}: {data}")
    except WebSocketDisconnect:
        manager.disconnect(ws)
        await manager.broadcast("A user left the room")

Error Handling & Exception Handlers

Raise HTTPException for expected client errors. Register custom exception handlers for app-level exceptions using @app.exception_handler(). FastAPI's default 422 validation error response can also be overridden.

HTTPException

HTTPException(status_code, detail, headers) — detail can be a string, dict, or any JSON-serialisable value. It appears in the "detail" key of the response body.

Custom exception classes

Define domain-specific exceptions (e.g. ItemNotFoundException) and register a handler. Keeps routes clean and centralises error formatting.

⚠ Validation errors (422)
FastAPI automatically catches Pydantic ValidationError and returns a structured 422 with per-field error details. Override RequestValidationError handler to customise the format.
code
from fastapi import FastAPI, HTTPException, Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

app = FastAPI()

# ── Standard HTTPException ────────────────────────────────────────────────────
@app.get("/items/{item_id}")
async def get_item(item_id: int):
    item = db.get(item_id)
    if not item:
        raise HTTPException(
            status_code=404,
            detail={"message": "Item not found", "item_id": item_id}
        )
    return item

# ── Custom exception class ────────────────────────────────────────────────────
class EntityNotFoundError(Exception):
    def __init__(self, entity: str, id: int):
        self.entity = entity
        self.id = id

@app.exception_handler(EntityNotFoundError)
async def not_found_handler(request: Request, exc: EntityNotFoundError):
    return JSONResponse(
        status_code=404,
        content={"error": "NOT_FOUND", "entity": exc.entity, "id": exc.id}
    )

@app.get("/users/{id}")
async def get_user(id: int):
    raise EntityNotFoundError("User", id)   # handled globally

# ── Override default 422 validation error format ──────────────────────────────
@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    errors = [
        {"field": ".".join(str(l) for l in e["loc"][1:]), "msg": e["msg"]}
        for e in exc.errors()
    ]
    return JSONResponse(
        status_code=422,
        content={"error": "VALIDATION_ERROR", "fields": errors}
    )

Testing with pytest

FastAPI's TestClient wraps httpx and runs your app in-process — no server needed. It also supports async tests via AsyncClient. Swap dependencies using app.dependency_overrides for clean, isolated tests.

Testing strategy

  • Use a test database — separate SQLite or an in-memory DB, never touch production
  • Override get_db — inject a test session that rolls back after each test
  • Use fixtures — conftest.py for app, client, and DB session setup
  • Test the contract — status codes, response shape, error cases
💡 Async tests
For async route tests use httpx.AsyncClient with ASGITransport. Install pytest-asyncio and mark tests with @pytest.mark.asyncio.
code

Deployment & Production

For production, run FastAPI with Gunicorn managing Uvicorn worker processes. Gunicorn handles process management and restarts; Uvicorn handles the async ASGI requests.

Gunicorn + Uvicorn workers

The worker count formula: 2 × CPU cores + 1. For an 8-core server, use --workers 17. More workers = more memory; tune for your instance.

Configuration checklist for production

  • Set docs_url=None, redoc_url=None in FastAPI() to disable public docs
  • Load secrets from environment variables — never hardcode them
  • Use pool_pre_ping=True on your DB engine to detect stale connections
  • Add TrustedHostMiddleware to prevent host header injection
  • Set a reverse proxy (nginx) in front to handle TLS, rate limiting, static files
  • Use structured JSON logging — log request ID, method, path, status, duration

Docker — minimal production image

--no-cache-dirReduce image size — no pip cache needed in containers
COPY only requirements firstLayer caching — rebuild app layer, not deps layer, on code change
Non-root USERAlways run as non-root for security
